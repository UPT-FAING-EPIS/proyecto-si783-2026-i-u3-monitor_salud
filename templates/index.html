Integraciones

API REST · Agentes IA · VS Code

Consume los datos del monitor desde tus propias aplicaciones, integra la Skill en agentes como Codex, Antigravity o Copilot, y usa la extensión nativa de VS Code.

ℹ️
Base URL: http://tu-servidor:5000/api/v1 — Autentícate con Bearer Token o sesión de navegador.

🚀 Inicio rápido

1
Crea una API Key
Ve a la pestaña API Keys y genera un token para tu aplicación. El token comienza con dhm_.
2
Autentica tus peticiones
Incluye el header Authorization: Bearer <tu_api_key> en todas tus peticiones.
bash 
curl -H "Authorization: Bearer dhm_TU_API_KEY" \
  http://tu-servidor:5000/api/v1/health
3
Consulta métricas en tiempo real
Obtén el estado de todas tus bases de datos en una sola petición.
bash 
curl -H "Authorization: Bearer dhm_TU_API_KEY" \
  http://tu-servidor:5000/api/v1/summary
4
Ejemplo con Python
Integra el monitor en tus scripts de automatización o dashboards externos.
python 
import requests

API_KEY = "dhm_TU_API_KEY"
BASE_URL = "http://tu-servidor:5000/api/v1"
headers = {"Authorization": f"Bearer {API_KEY}"}

# Resumen global
r = requests.get(f"{BASE_URL}/summary", headers=headers)
data = r.json()
print(f"Bases: {data['total_datasources']} | Online: {data['online']}")

# Métricas por datasource
r = requests.get(f"{BASE_URL}/metrics", headers=headers)
metrics = r.json()
for ds_id, entry in metrics.items():
    m = entry.get("metrics", {})
    print(f"DS {ds_id}: CPU={m.get('cpu_pct')}% Cache={m.get('cache_hit_ratio')}%")

Skill oficial del Monitor de Salud DB

Esta skill le permite a cualquier agente de IA (Codex, Antigravity, Copilot, Cursor, etc.) interactuar de forma autónoma con tus bases de datos: consultar métricas, revisar alertas, leer logs y más — todo usando la API REST o el CLI local de la extensión.

Descargar SKILL.md

🚀 Cómo integrar la Skill en tu agente

1
⚠️ Obtén tu API Token primero
🔑
Antes de configurar cualquier agente, ve a la pestaña API Keys de esta sección y genera un token personal. El token tiene el formato dhm_xxxxxxxxxxxx. Cópialo inmediatamente — el sistema no lo volverá a mostrar completo.
2
Descarga el archivo SKILL.md
El repositorio incluye el manifiesto de la skill en .agents/skills/db-health-monitor/SKILL.md. Este archivo contiene el frontmatter YAML con nombre y descripción que los agentes leen automáticamente, además de instrucciones detalladas de uso.
yaml
---
name: db-health-monitor
description: >
  Interactúa con el Monitor de Salud DB para listar bases
  de datos, consultar métricas en tiempo real, obtener
  historiales, leer logs y auditar alertas. Usa la API REST
  con tu Bearer Token o el CLI local (helper.py).
---
3
Úsala con Codex CLI
Si usas OpenAI Codex CLI, coloca el archivo en ~/.codex/skills/ o referéncialo en tu codex.yaml. El agente detectará automáticamente la skill y la usará para responder preguntas sobre el estado de tus bases de datos.
bash
# Instalar la skill en Codex
cp .agents/skills/db-health-monitor/SKILL.md ~/.codex/skills/db-health-monitor.md

# Consultar el estado de tus bases de datos
codex "¿Cuáles son las bases de datos registradas y su estado?"
4
Úsala con Antigravity / Cursor / Copilot
Para agentes basados en workspace (como Antigravity IDE, Cursor o GitHub Copilot Workspace), basta con tener el directorio .agents/ en la raíz del repositorio. El agente detecta y carga la skill automáticamente al abrir el proyecto.
bash
# La estructura ya está lista en el repo:
.agents/
  skills/
    db-health-monitor/
      SKILL.md   ← detectado automáticamente
5
Ejemplo: script Python para agentes
Este patrón muestra cómo un agente puede usar la skill para obtener métricas reales de forma autónoma vía API REST.
python
import requests

# Token generado en la pestaña API Keys
API_KEY = "dhm_TU_TOKEN_AQUI"
BASE_URL = "http://38.250.116.71:5000"
headers = {"Authorization": f"Bearer {API_KEY}"}

# 1. Listar todas las bases de datos registradas
ds = requests.get(f"{BASE_URL}/api/datasources", headers=headers).json()
for db in ds:
    print(f"{db['nombre']} ({db['tipo_db']}) → {db['status']}")

# 2. Obtener métricas de la primera base de datos
if ds:
    ds_id = ds[0]['id']
    m = requests.get(f"{BASE_URL}/api/metrics?datasource_id={ds_id}", headers=headers).json()
    print(f"CPU: {m.get('cpu_pct')}% | Cache: {m.get('cache_hit_ratio')}%")

# 3. Revisar alertas recientes
alertas = requests.get(f"{BASE_URL}/api/alerts", headers=headers).json()
for a in alertas[:5]:
    print(f"[{a['severity']}] {a['message']}")

Panel lateral premium con glassmorphism, indicadores LED en tiempo real, dashboard de métricas con gráficas suavizadas y auditoría de seguridad SQL integrada directamente en tu editor.

Descargar del Marketplace Abrir en VS Code

⚙️ Configuración de la extensión

1
⚠️ Genera tu API Token primero
🔑
Ve a la pestaña API Keys y genera un token personal antes de configurar la extensión. El token tiene el formato dhm_xxxxxxxxxxxx. Guárdalo ahora — no podrás verlo completo de nuevo.
2
Instala la extensión
Usa los botones de arriba para instalarlo desde el Marketplace, o bien desde el VSIX compilado:
shell
code --install-extension vscode/db-health-monitor-0.1.5.vsix
3
Abre la Paleta de Comandos y conecta
En VS Code, presiona Ctrl+Shift+P y ejecuta DHM: Conectar al Servidor. Ingresa:
  • Python Path: ruta a tu intérprete Python con dependencias instaladas
La extensión lee el config.ini del workspace actual automáticamente.
4
Explora el Panel Lateral
Haz clic en el ícono Monitor de Salud DB en la Barra de Actividad. Verás:
  • 🟢 Pestaña Bases de Datos: tarjetas con LED neón de estado en tiempo real y botón «Ver Dashboard»
  • 🔍 Pestaña Auditoría SQL: escaneo recursivo de archivos .sql con detección de inyecciones, credenciales expuestas y privilegios peligrosos — con navegación directa a la línea exacta del problema
